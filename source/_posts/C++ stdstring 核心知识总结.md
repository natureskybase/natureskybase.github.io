### 1. `std::string` 的内存分配：`resize + 下标索引` 与 `reserve + push_back`

**`std::string` 内部维护两个核心状态值：**

- **`size()`**：当前实际存储的有效字符数（即字符串长度）。
- **`capacity()`**：当前已申请的底层堆内存空间大小（能容纳的字符数，`capacity >= size`）。

**`std::string` 两个核心内存分配方法：**

- **`resize(n)`**:修改 `size`，如果 `n > capacity` 则扩容，并将字符填充为默认的`'\0'`，并且可以像数组一样使用索引进行访问。
- **`reserve(n)`**:不会修改`size`，仅申请裸内存，不构造字符；不能使用索引进行访问。取而代之可以使用**`push_back(c)`**的方法自动将苏剧填充到尾部。

------

- **场景 A（已知确切长度，需随机访问修改）**：使用 **`resize(n) + 下标索引`**。
  
  ```cpp
  string result;
  result.resize(word1.size() + word2.size()); // 一次性分配并填充 '\0'
  for (int i = 0; i < len; ++i) {
      result[2*i] = word1[i];   // 安全，因为 size 已足够
      result[2*i+1] = word2[i];
  }
  ```
  
- **场景 B（已知确切长度或大概长度，顺序追加）**：使用 **`reserve(n)` + `push_back`**（**最优性能，推荐**）。
  
  ```cpp
  string result;
  result.reserve(total_len); // 只申请内存，不初始化，无 '\0' 写入开销
  for (...) {
      result.push_back(word1[i]); // 追加，size 自动增加，且因容量足够不触发重分配
      result.push_back(word2[i]);
  }
  // 追加剩余部分
  result.append(word1.begin() + minLen, word1.end());
  ```

**Note**：

- `resize` 会做 **“初始化写入”**（多余的 `'\0'` 赋值），如果随后你马上用 `[]` 覆盖所有值，这步初始化是性能浪费。
- `reserve` 只做 **“内存申请”**，搭配 `push_back` 避免了无效写入，是 C++ 中构造动态字符串的最优解。

### 2. `std::string` 的内存特性

| 特性                   | 说明                                                         |
| :--------------------- | :----------------------------------------------------------- |
| **RAII 资源管理**      | 构造函数分配内存，析构函数自动释放内存。无需手动 `free`/`delete`，**彻底杜绝内存泄漏**。 |
| **连续内存保证**       | C++11 起，`std::string` 的数据在内存中严格连续，可通过 `&str[0]` 或 `str.data()` 获取底层 `char*`。 |
| **短字符串优化 (SSO)** | 对于极短的字符串（通常 ≤ 15 字节），`std::string` 会直接存储在对象栈空间内，**不进行堆分配**，大幅提升小字符串性能。 |
| **边界安全选项**       | `operator[]`（无边界检查，与 C 数组一样快）；`.at(idx)`（有边界检查，越界抛 `std::out_of_range` 异常）。 |
| **高效的移动语义**     | 返回 `std::string` 时，依赖移动构造函数（C++11 起）和返回值优化（RVO），避免了深拷贝。 |

### 3. 与 C 语言 `char*` / 数组的深度对比

| 对比维度          | **C++ `std::string`**                                        | **C 风格 `char arr[N]`（栈数组）**                           | **C 风格 `char *ptr = malloc(N)`（堆指针）**                 |
| :---------------- | :----------------------------------------------------------- | :----------------------------------------------------------- | :----------------------------------------------------------- |
| **内存位置**      | 对象在栈，数据通常在堆（SSO 优化除外）。                     | 全部在**栈**上（或静态存储区）。                             | 指针在栈，数据在**堆**上。                                   |
| **生命周期**      | **自动**（析构函数释放堆内存）。                             | **自动**（栈弹出即销毁）。                                   | **完全手动**（必须调用 `free`）。                            |
| **空状态安全**    | 空字符串 `string s;` 立即有效，`s.c_str()` 返回 `""`。       | `char arr[10];` 内容为**垃圾值**，需手动 `memset(arr, 0, sizeof(arr));`。 | `char *p = NULL;` 解引用立即崩溃；必须手动 `malloc`。        |
| **长度/容量跟踪** | 内置 `size()` 和 `capacity()`，高效 O(1) 获取。              | 需手动 `strlen` 遍历（O(N)）或用宏定义固定长度。             | 需手动记录长度变量，极其繁琐。                               |
| **动态扩容**      | **全自动**。超出容量时自动 `realloc`（且保证数据完整）。     | **不支持**。长度固定，超界即栈溢出。                         | **手动**。用 `realloc`，需处理旧指针失效，极易出错。         |
| **拷贝/赋值代价** | `s2 = s1` 默认**深拷贝**（复制所有字符），或使用 `std::move` 窃取资源（零拷贝）。 | 数组名退化为指针，`arr2 = arr1` 语法错误，必须用 `strcpy`/`memcpy`。 | 指针赋值 `p2 = p1` 仅是**浅拷贝**（两个指针指向同一内存），极易造成悬垂指针和双重释放。 |
| **越界访问**      | `operator[]` **不检查**（与 C 一致，快）；`.at()` 会检查（安全）。 | **不检查**，越界破坏栈帧，隐蔽性极强。                       | **不检查**，越界破坏堆元数据，`free` 时崩溃。                |
| **字符串拼接**    | `s += "World"` 或 `s.append("!")`，自动管理长度。            | `strcat(dest, src)` 需确保 dest 缓冲区足够，否则溢出。       | 需手动 `realloc` 扩大空间再 `strcat`，极其痛苦。             |
